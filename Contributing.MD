# Contributing to Awesome Engineering AI

Thank you for your interest in contributing!

## Guidelines

### Adding a new tool or resource

1. Make sure the tool/resource is related to **AI for physical/hardware engineering** (not pure software dev).
2. Check that it's not already listed.
3. Add your item in the appropriate category, in alphabetical order within the section.
4. Use the following format:

```markdown
- [Name](https://link) - Brief description. What makes it unique or useful.
```

5. Keep descriptions concise (1-2 sentences max).
6. Submit a Pull Request with a clear title explaining what you're adding.

### Suggesting a new category

If you think a new category is needed, open an Issue first to discuss it.

### Quality standards

- **Active projects only**: No abandoned repos with no activity in 12+ months (unless **historically** significant).
- **Working links**: Verify all URLs are accessible.
- **No duplicates**: Check existing entries before adding.
- **Engineering focus**: Tools must have clear application in physical/hardware engineering workflows.

### What we're especially looking for

- Open-source engineering AI tools
- MCP servers for CAD/CAE/CAM
- AI research papers applied to physical engineering
- Regional or niche tools not well-known in English-speaking communities
- Claude Code skills/plugins for engineering workflows

## Code of Conduct

Be respectful. This is a community resource for engineers building real things.
